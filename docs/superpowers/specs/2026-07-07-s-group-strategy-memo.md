## S.0 존재 이유와의 정합

OnchainAI 본질 = "디파이 환경에서 AI agent가 잘 활동하도록 툴을 모아둔 곳". 디파이는 코드 분산·자금 추적 어려움·일일 수조사. 전략 상품은 이 페인을 **"정리된 자금흐름 DAG + 남은 수동 TODO"**로 치환 — 디렉터리(재료 목록)에서 전략(레시피+조리순서+위험표)으로 상향. 옴니사이언스 아님, **갭 서피싱이 기능**.

## S.2 S0 gap_audit 설계 (첫 상품)

- **입력**: 인텐트(자연어) → 서브골 분해(인텐트 분해 로직만, DAG 합성 없음).
- **출력**: 서브골별 "카탈로그에 있음(후보 slug 나열) / 갭(없음, 수동 조사 필요)" 표. 자금흐름 그래프·실행 순서는 **포함 안 함**(그게 S1).
- **가치**: "뭘 모아야 할지"부터 알려줌. 디파이 수조사 부담의 첫 단계 해소.
- **정직**: 커버리지 상한 명시 — 갭이 많으면 "현재 카탈로그 범위 한계"로 고지. 할루 0(실재 slug만).
- **인프라**: `search_tools` + function/chain taxonomy만. 신규 = 인텐트 분해 로직. DAG/리스크 주석 없음.
- **수용 기준**: 모든 "있음" 인용이 실재 slug / 갭이 속김 없이 명시 / disclaimer+타임스탬프.

## S.3 S1 전략 생성 메커니즘 (S0 검증 후)

**intent → 서브골 분해 → 카탈로그 매핑 → 자금흐름 DAG 합성 → 툴 검증 → 리스크+갭 주석 → 납품**

예시("Base native BTC → Morph 루핑 자동화"):

1. **인텐트 분해**: ① BTC→Base ② Morph 예치 ③ 담보대출 ④ 차입금 재스왑/재예치(루프) ⑤ 자동화 에이전트 ⑥ 청산 모니터.
2. **서브골별 카탈로그 매핑**: `search_tools`(function/chain 필터). **모든 추천이 카탈로그 실제 툴 인용**(할루 없음). 없으면 "갭".
3. **자금흐름 DAG 합성**: 툴을 실행 순서 그래프로 연결 → **Blueprint**(edges 재사용). 노드=툴·간선=자금 이동. 흩어진 수조사를 하나의 흐름도로 시각화.
4. **툴 검증(verifyable만)**: A의 `probe_x402_endpoint`로 liveness+요금. trust tier·install_risk 표시. **툴 진짜/살아있음은 증명, 전략 수익성은 증명 안 함**.
5. **리스크+갭 주석**: 노드마다 청산 리스크(LTV/청산임계)·브릿지 리스크·가스·x402 요금(검증)·"실행 미검증"·"카탈로그에 없어 수동 조사 필요". **갭 서피싱 = 버그 아닌 기능**.
6. **납품**: Blueprint(자금흐름 DAG) + 노드별 설치가이드 + 리스크/갭 표 + (선택) 실행 스킬 파일(S2).

## S.4 정직 설계 / 한계 (snake oil 방지)

- **옴니사이언스 아님**: 툴 메타데이터 수준 지식. 컨트랙트 내부 로직 미인덱스. 전략 = "plausible 아키텍처 스캐폴드"지 수익 보증 아님. 강한 disclaimer("교육적 정보, 자체 검증 책임, 금융 조언 아님").
- **카탈로그 커버리지 = 품질 상한**: 초기엔 다수 서브골이 "갭·수동 필요". **W1 Bazaar 승인이 S0/S1의 선행 조건**.
- **신선도**: 크롤러+L4 프로브로 툴 liveness fresh, 프로토콜 로직 변경은 미추적 → "시각 T 기준" 표기.
- **경로 최적화 한계(S3)**: 견적 API 노출 툴만 가격 최적화 가능, 없으면 툴 수준 라우팅. 범위 명시.

## S.5 인프라 재사용 (신규 배관 최소)

- `search_tools`(서브골 매핑) + function/chain taxonomy + trust/install_risk + Product A `probe_x402_endpoint`(검증) + Blueprint edges(DAG) + Agent Sync/plugin(스킬 납품).
- 신규: 인텐트 분해 로직(S0에 먼저) → DAG 합성 + 리스크/갭 주석 규칙(S1) + 갭 서피싱 포매터.

## S.6 OKX 등재 연동

- S1/S4 → **OKX A2A Task 마켓**(인간 태스크 → agent 납품 → 온체인 에스크로 정산, 비커스토디얼). 자사 수취라 하드룰 OK. A2A는 S-group에 한해 in-scope(infra-waves §2). infra-waves §3.2(A2MCP 표)와 본 S.1 표(A2A) 분리 유지.
- S2 → 자사 스킬 마켓 + OKX에 "OnchainAI Strategist" 에이전트로 노출.
- S0/S3/S6 → A2MCP/x402 per-call.
